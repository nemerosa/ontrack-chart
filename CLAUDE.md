## Overview

This is a Helm 3 chart repository that deploys **Ontrack** (a CI/CD tracking platform) to Kubernetes. The chart is published to Docker Hub as an OCI artifact at `oci://registry-1.docker.io/nemerosa/yontrack-chart`.

The chart bundles three sub-charts: PostgreSQL 17, RabbitMQ 4, and Elasticsearch 9 (Bitnami).

## Common Commands

```bash
# Lint the chart
helm lint charts/ontrack

# Render templates locally (for inspection)
helm template ontrack charts/ontrack -f values.yaml > template.yaml

# Validate Docker images referenced in the chart are reachable
./scripts/check-images.sh

# Regenerate values documentation (requires helm-docs)
helm-docs charts/ontrack

# Package the chart
helm package charts/ontrack

# Build Keycloak theme tarball (required before packaging)
tar -czf charts/ontrack/files/themes/yontrack.tar.gz -C charts/ontrack/files/themes/yontrack .
```

## Architecture

### Chart Structure

The main chart lives in `charts/ontrack/`. Key files:

- `Chart.yaml` — chart metadata; `appVersion` is auto-updated by Ontrack's auto-versioning system via PRs
- `values.yaml` — all configurable defaults (825 lines); all values are documented inline for `helm-docs`
- `templates/_helpers.tpl` — shared template helpers (naming, labels, secret generation)
- `files/themes/` — custom Keycloak login theme (packaged as `yontrack.tar.gz`)
- `files/partials/keycloak/` — Keycloak realm/client JSON templates

The root-level `values.yaml` is a local override file (gitignored) used for local testing; the canonical defaults are in `charts/ontrack/values.yaml`.

### Deployed Components

| Component | Template prefix | Notes |
|-----------|----------------|-------|
| Ontrack backend | `ontrack-*` | StatefulSet with PVC for data |
| Next.js UI | `ui-*` | Frontend, uses NextAuth |
| PostgreSQL | sub-chart | Ontrack's main database |
| RabbitMQ | sub-chart | Messaging |
| Elasticsearch | sub-chart | Search indexing |
| Keycloak | `keycloak-*` | Optional; has its own PostgreSQL StatefulSet |

### Authentication Modes

Authentication is controlled via `auth.*` values:

- **Keycloak (default)**: Deploys a bundled Keycloak instance with its own PostgreSQL. Supports LDAP federation and comes with a custom `yontrack` theme.
- **External OIDC**: Configure via `auth.oidc.*` for Okta, Auth0, Microsoft Entra, etc. See `docs/` for provider-specific setup.
- **LDAP**: Direct LDAP or via Keycloak federation (`auth.keycloak.ldap.*`).

### Secrets Management

Templates exist in pairs for many secrets: a plain Kubernetes secret (`*-secret.yaml`) and an ExternalSecret variant (`*-external-secret.yaml`). The ExternalSecret variant integrates with the [External Secrets Operator](https://external-secrets.io/) for pulling credentials from Vault, AWS Secrets Manager, etc. Use `auth.keycloak.*.useExternalSecret` / similar flags to switch between them.

### CI/CD

`.github/workflows/release.yml` runs on every push to `main`:
1. Prepares Keycloak theme tarball
2. Lints and validates Docker image availability
3. Packages the chart
4. Detects whether `chart.version` changed since the previous run
5. If new version: pushes OCI artifact to Docker Hub and creates a GitHub release with a generated changelog

`.yontrack/ci.yaml` configures Ontrack's auto-versioning: when a new Ontrack release is promoted upstream, a PR is automatically opened updating `appVersion` in `Chart.yaml`.

### Documentation Generation

`charts/ontrack/README.md` is auto-generated by `helm-docs` from inline comments in `values.yaml`. Do not edit it manually. When modifying values, update the corresponding `# -- description` comment above the value.
